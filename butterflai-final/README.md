# 🦋 ButterflAI — End-to-End ML Training Platform

> Type one sentence. Get a trained model. No DevOps. No PhD.

**Stack:** Next.js 14 · Vercel · Google OAuth + Drive · Supabase · Modal GPU · Gemini/Groq/Claude AI

---

## What it does

1. **Describe** — you type one sentence about your ML goal
2. **Discover** — AI searches Kaggle + HuggingFace, ranks datasets by relevance (cached 7 days in Supabase)
3. **Configure** — AI-prefilled hyperparams; you adjust if needed
4. **Validate & Auto-Fix** — AI checks 10 consistency dimensions, auto-rewrites broken code sections with full diff + reason for each change
5. **Train Live** — Modal T4 GPU trains the model; SSE streams logs + metrics to your browser in real time
6. **Deploy** — Download `best_model.pth`, get `streamlit_app.py`, open Drive folder

## Quick Start (10 min)

### 1. Clone & install
```bash
git clone https://github.com/you/butterflai
cd butterflai
bash setup.sh   # installs deps, creates .env.local
```

### 2. Create Supabase project (free)
1. Go to [supabase.com](https://supabase.com) → New project
2. Settings → API → copy URL + anon key + service role key
3. SQL Editor → New query → paste `supabase_schema.sql` → Run

### 3. Google OAuth + Drive
1. [console.cloud.google.com](https://console.cloud.google.com) → New project
2. APIs & Services → Library → enable **Google Drive API**
3. APIs & Services → Credentials → Create → OAuth 2.0 Client ID → Web
4. Authorized redirect URIs:
   - `http://localhost:3000/api/auth/callback/google`
   - `https://your-app.vercel.app/api/auth/callback/google`
5. Copy Client ID + Secret

### 4. AI provider keys (all free)
| Provider | Where | Free tier |
|----------|-------|-----------|
| **Gemini Flash** | [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey) | 15 RPM, 1M TPM |
| **Groq Llama 3** | [console.groq.com](https://console.groq.com) | 14,400 req/day |
| **Claude Haiku** | [console.anthropic.com](https://console.anthropic.com) | Paid but cheapest |

### 5. Modal GPU setup
```bash
pip install modal
modal token new                              # authenticates you
modal secret create butterflai-secrets \
  KAGGLE_USERNAME=your_user \
  KAGGLE_KEY=your_key
modal deploy modal/butterflai_worker.py     # deploys GPU function (one-time)
```

### 6. Fill .env.local
```bash
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<already generated by setup.sh>
GOOGLE_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=xxx
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJxxxx
SUPABASE_SERVICE_ROLE_KEY=eyJxxxx
GEMINI_API_KEY=AIzaxxxx
GROQ_API_KEY=gsk_xxxx
ANTHROPIC_API_KEY=sk-ant-xxxx   # optional
MODAL_TOKEN_ID=ak-xxxx
MODAL_TOKEN_SECRET=xxxx
```

### 7. Run locally
```bash
npm run dev
# Open http://localhost:3000
```

### 8. Deploy to Vercel
```bash
npx vercel --prod
# Add all env vars in Vercel dashboard → Settings → Environment Variables
```

## Architecture

```
Browser (Next.js)
│
├─ /api/auth/[...nextauth]  Google OAuth, Drive scope, token refresh
├─ /api/analyze             Multi-AI (Gemini/Groq/Claude) + Supabase cache
├─ /api/validate            Code gen + consistency check + auto-fix + diff
├─ /api/jobs                Job CRUD + model output storage
├─ /api/drive-upload        Upload files to user's Google Drive
└─ /api/train               Modal.com SSE stream ─────────────────────────┐
                                                                           │
                                        Modal T4 GPU Worker ───────────────┘
                                        ├─ Download Kaggle / HuggingFace dataset
                                        ├─ Run train.py (streams stdout)
                                        ├─ Generate streamlit_app.py
                                        └─ Upload all outputs → Drive
```

## Database (Supabase)

| Table | What's stored |
|-------|--------------|
| `profiles` | User settings, preferred AI, job count |
| `jobs` | Every training run — config, status, accuracy, Drive links |
| `model_outputs` | Drive file IDs for model, Streamlit app, history |
| `consistency_reports` | Before/after code, fixes applied, diff |
| `dataset_cache` | Query → datasets, 7-day TTL, hit counter |
| `training_epochs` | Per-epoch metrics for all jobs |

## Consistency Auto-Fix

When the AI detects issues between your dataset, config, and code:

1. **Detect** — 10 checks: format, class alignment, GPU memory, augmentation, loss function, transfer learning, convergence, goal alignment, data size, image size
2. **Fix** — AI rewrites only the broken sections of `train.py`
3. **Re-validate** — runs checks again to confirm issues resolved
4. **Explain** — every fix shows: what changed, before/after code, why it was wrong

Example fixes:
- Batch size `64` → `32` when EfficientNet-B3 at 384px would OOM on T4
- Wrong loss function for multi-label → BCE instead of CrossEntropy
- Missing normalization for HuggingFace dataset format
- Image size mismatch with ViT patch size requirements
